HTML: HyperText Markup Language

HTML is the standard markup language used to create webpages and structure content on the internet. It forms the foundation of every website by organising text, images, links, forms, tables, and multimedia into a format that browsers can display.

What Is HTML?

HTML uses elements and tags to define different parts of a webpage, such as headings, paragraphs, buttons, images, links, forms, and navigation menus. It gives content meaning and structure, while CSS controls the appearance and JavaScript adds interactivity.

Modern web development relies on HTML because it helps developers create organised, accessible, and browser-compatible webpages that work across phones, tablets, laptops, and desktop computers.

Structure Of An HTML Document

Every HTML document follows a basic structure. The <head> contains metadata and links to resources, while the <body> contains the visible content users see on the page. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
    <link rel="stylesheet" href="css/style.css">
  </head>
  <body>
    <header>Site header</header>
    <main>
      <h1>Hello, world!</h1>
      <p>Page content goes here.</p>
    </main>
    <footer>Site footer</footer>
    <script src="js/script.js" defer></script>
  </body>
</html>

Boilerplate Explanation

DOCTYPE

<!DOCTYPE html> tells the browser to use modern standards mode when rendering the page.

HTML Element

<html lang="en"> is the root element. The lang attribute supports accessibility and search engines by declaring the page language.

Head Section

The <head> stores metadata, the page title, character encoding, viewport settings, and stylesheet links.

Body Section

The <body> contains all visible webpage content, including headings, paragraphs, images, buttons, forms, and links.

External Files

HTML can connect to CSS using <link> and to JavaScript using <script>, allowing structure, design, and behaviour to work together.

Semantic HTML

Semantic HTML means using meaningful tags that describe the purpose of the content. Examples include <header>, <nav>, <main>, <section>, <article>, and <footer>. 

<header>
  <nav>Main navigation</nav>
</header>
<main>
  <section>
    <article>
      <h2>Article Title</h2>
      <p>Independent content goes here.</p>
    </article>
  </section>
</main>

<footer>Copyright and contact information</footer>

Semantic HTML improves accessibility, search engine optimisation, code readability, and website organisation because both humans and browsers can understand the role of each section.

HTML Attributes

Attributes provide extra information about HTML elements. They appear in the opening tag as name-value pairs, such as href="index.html" or class="card".

Attribute Used With Purpose
href <a>, <link> Defines the target URL for a link or external resource.
src <img>, <script> Defines the file or resource to load.
alt <img> Provides alternative text for images and supports screen readers.
id Global Gives an element a unique identifier.
class Global Assigns class names for CSS styling and JavaScript selection.
type <input>, <button> Defines the input or button behaviour.
required Form controls Makes a form field mandatory before submission.
for <label> Connects a label to a form control using the control's ID.

HTML Form Validation Demo

HTML can perform basic form validation without JavaScript. Attributes such as required, type="email", and minlength help browsers check user input before a form is submitted.

If the email is not valid or the password is shorter than eight characters, the browser can show a validation message. This demonstrates built-in HTML validation features. 

<input type="email" required placeholder="Enter Email">

<input type="password" minlength="8" required>

Accessibility Demo

Accessibility means making websites usable for as many people as possible. The alt attribute is important because screen readers use it to describe images to visually impaired users.

Bad Example

<img src="cat.jpg">

This image has no alternative text, so users with screen readers may not understand what the image shows.

Good Example

<img src="cat.jpg" alt="Orange cat sleeping on a chair">

This image includes useful alternative text, making the content more accessible and meaningful.

Images, Links, And Multimedia

Images

The <img> element displays images. The alt attribute gives alternative text when the image cannot load and helps screen reader users understand the image.

Links

The <a> element creates clickable links. Links help users move between pages, open resources, or jump to sections of the same page.

Multimedia

HTML5 supports media elements such as <video> and <audio>, allowing websites to include media without relying on external plugins.

Tables And Lists

HTML tables and lists help organise information clearly. Lists are useful for grouped points or steps, while tables are best for structured data with rows and columns.

Accessibility In HTML

Accessibility means designing websites so that all users, including people with disabilities, can access and use them effectively. HTML supports accessibility through semantic structure, alt text, form labels, meaningful links, and correct heading order.

Alt Text

Alternative text describes images for screen readers and appears if an image fails to load.

Labels

Labels identify form fields clearly and make forms easier to use with assistive technologies.

Keyboard Navigation

Good HTML allows users to move through links, buttons, and form fields using a keyboard. This is important for users who cannot use a mouse.

Heading Order

Headings should follow a clear order, such as h1, h2, and h3. This helps screen readers and users understand the page structure.

Common HTML Mistakes

Avoiding common mistakes helps make webpages easier to read, maintain, and use.

Missing Alt Text

Images should include useful alt text so users with screen readers can understand the image content.

Incorrect Heading Order

Skipping heading levels can make the page structure confusing for users and assistive technologies.

Unclosed Tags

Missing closing tags can break the layout and cause unexpected page behaviour.

Using Divs For Everything

Semantic tags should be used where possible instead of relying only on generic div elements.

Conclusion

HTML is the foundation of web development. It gives webpages structure, meaning, and accessibility. When HTML is written clearly with semantic elements, correct attributes, forms, tables, lists, and media, it becomes easier to style with CSS and make interactive with JavaScript.
